# DevFocus 운영 배포 가이드

## 현재 연결 정보

- Frontend: Vercel Git 연동 방식
- Backend 예상 주소: `http://13.125.181.228:3001`
- Vercel API 호출: `/backend-api/*`를 통해 AWS로 서버 프록시
- 운영 비밀번호와 API Secret은 Git에 저장하지 않는다.

## Vercel 자동 배포

1. GitHub에 이 저장소를 생성하고 `main` 브랜치를 push한다.
2. Vercel에서 `Add New Project`로 저장소를 연결한다.
3. Root Directory를 `P2_코드/frontend`로 지정한다.
4. 환경변수를 확인한다.
   - `NEXT_PUBLIC_API_URL=/backend-api`
   - `BACKEND_ORIGIN=http://13.125.181.228:3001`
   - `NEXT_PUBLIC_TOSS_CLIENT_KEY=토스_클라이언트_키`
5. Production Branch를 `main`으로 설정한다.

이후 GitHub `main` 브랜치에 push하면 Vercel이 자동 재배포한다.

## AWS 백엔드

AWS 보안 그룹 인바운드에 최소 다음 포트가 필요하다.

- `22/TCP`: 관리자 IP에서만 허용
- `3001/TCP`: Vercel 프록시 연결을 위해 허용

서버의 `/opt/devfocus/.env.production`은 `.env.production.example`을 기준으로 생성한다. 서버 로그인 비밀번호를 `DB_PASSWORD`로 재사용하면 안 된다.

```bash
docker compose --env-file .env.production -f docker-compose.production.yml up -d --build
curl http://localhost:3001/health
```

## Toss Payments 실결제

1. Toss Payments 개발자센터에서 상점 계정을 만든다.
2. 결제위젯 연동 키의 테스트 키로 결제 E2E를 먼저 검증한다.
3. Vercel에 `NEXT_PUBLIC_TOSS_CLIENT_KEY`를 등록한다.
4. AWS `.env.production`에 같은 상점의 `TOSS_SECRET_KEY`를 등록한다.
5. Toss에 Vercel 운영 도메인과 성공/실패 URL을 등록한다.
6. 테스트 결제 승인·영수증·취소·수강 등록을 확인한다.
7. 심사 완료 후 두 키를 운영 키로 함께 교체한다.

Client Key는 프론트에 노출 가능한 키지만 Secret Key는 AWS 서버에만 저장해야 한다. 테스트 키와 운영 키를 섞으면 승인에 실패한다.
