# DevFocus 배포 복구 가이드

이 문서는 이전 배포 코드와 설정을 잃어버린 상태에서 DevFocus를 다시 복구하는 절차를 정리한 것이다.

## 1. 소스 복원

저장소 전체 복제용 번들을 먼저 푼다.

```bash
git clone /path/to/devfocus.bundle devfocus
cd devfocus
git log --oneline -1
```

번들에는 다음이 포함되어야 한다.

- `AGENTS.md`, `CODEX.md`, `.codex`
- `P2_코드/frontend`, `P2_코드/backend`
- 발표 자료와 제출 문서
- 운영용 `docker-compose.production.yml`

## 2. 프론트 복구

Vercel에서 새 프로젝트를 만들고 다음을 지정한다.

- Root Directory: `P2_코드/frontend`
- Build Command: `npm run build`
- Install Command: `npm ci`
- Production Branch: `main`

환경변수:

- `NEXT_PUBLIC_API_URL=/backend-api`
- `BACKEND_ORIGIN=http://13.125.181.228:3001`
- `NEXT_PUBLIC_TOSS_CLIENT_KEY=...`

이후 `main` 브랜치에 push하면 자동 재배포된다.

## 3. 백엔드 복구

AWS EC2 또는 동일한 컨테이너 호스트에 접속한다.

```bash
cd /opt/devfocus
git pull --ff-only origin main
cp .env.production.example .env.production
vim .env.production
docker compose --env-file .env.production -f docker-compose.production.yml up -d --build
curl http://localhost:3001/health
```

필수 환경변수:

- `FRONTEND_URL`
- `DB_PASSWORD`
- `DB_ROOT_PASSWORD`
- `JWT_SECRET`
- `WEBHOOK_SECRET`
- `TOSS_SECRET_KEY`

## 4. 결제 복구

Toss Payments에서는 같은 상점 계정의 키 쌍을 써야 한다.

- 프론트: `NEXT_PUBLIC_TOSS_CLIENT_KEY`
- 백엔드: `TOSS_SECRET_KEY`

테스트 키로 먼저 승인/취소/환불을 확인하고, 운영 도메인을 등록한 뒤 운영 키로 교체한다.

## 5. 운영 확인

배포 후 확인 항목:

- 프론트 메인 페이지가 열린다.
- 로그인/회원가입이 된다.
- 강의 상세에서 결제 위젯이 열린다.
- 결제 후 수강 등록이 반영된다.
- 관리자에서 결제/환불/로그를 확인할 수 있다.
- `/health`와 `/metrics`가 정상 동작한다.
