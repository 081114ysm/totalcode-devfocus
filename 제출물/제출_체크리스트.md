# DevFocus 제출 체크리스트

## 제출 파일

- 전체 Git 저장소: 소스, 문서, `.codex`, `AGENTS.md`, `CODEX.md` 포함
- Clone용 Git bundle: `제출물/devfocus.bundle`
- 발표 자료: `제출물/DevFocus_발표자료.pptx`
- 발표 PDF: `제출물/DevFocus_발표자료.pdf`
- 발표 원고: `제출물/발표_원고.md`
- 배포 상태: `제출물/배포_상태.md`
- 복구 가이드: `제출물/복구_배포_가이드.md`

## Bundle 복원

```bash
git clone devfocus.bundle devfocus
cd devfocus
git log --oneline
```

## 제출 전 외부 작업

1. GitHub 등 원격 저장소 생성 후 `git remote add origin ...`, `git push -u origin main`
2. 배포 플랫폼에 MySQL과 Backend 배포
3. Frontend 배포 후 `NEXT_PUBLIC_API_URL`을 실제 API 주소로 설정
4. 토스페이먼츠 키와 SMTP·Discord secret 설정
5. PPT의 공개 배포 주소를 실제 주소로 교체
6. 이전 배포를 잃어버렸다면 `복구_배포_가이드.md`를 따라 재구성

비밀키와 실제 `.env` 파일은 제출하거나 Git에 커밋하지 않습니다.
